<h2> Final Submission Overview </h2>
In this final submission, I have implemented a fully-functional Closet Organiser App with new features, improved design, and seamless functionality based on all the provided files, including integration of outfits, persistence via local storage, carousel images, and the weather API for a complete and polished user experience. Below is a detailed breakdown of the work done:

1. Lab Work & Exercises
I have completed all the lab work and exercises as part of the course curriculum, building a deeper understanding of Android development, UI components, databases, and API integrations. These exercises helped me implement essential features such as CRUD operations, user interface components, API integration, and persistence.

2. Outfit Management
As part of the new functionality, users can now create outfits by combining different clothing items stored in their closet. This allows users to organize their wardrobe into outfit categories based on the season or any preferences they may have.

Users can create outfits from their clothing items, view them, and remove items from an outfit.

Outfit management features a simple interface that displays the items within each outfit, with the option to view individual clothing items in greater detail.

3. Persistence
The app now supports persistence with a local database, enabling users to:

Store clothing items and outfits even after closing the app.

Retrieve clothing items and outfits from local storage upon reopening the app.

The SQLite database handles the persistence of the user's closet data, ensuring all changes are saved and easily retrievable, providing a seamless experience.

4. Weather API Integration
To enhance the user experience, the app now integrates a Weather API to suggest clothing based on the current weather conditions in the user's location. The app retrieves weather data and provides recommendations for what to wear based on temperature, season, and user-defined preferences.

The Weather API provides real-time data, which is used to suggest appropriate outfits for the day.

This feature helps users make decisions about which clothing items to wear, ensuring they are dressed appropriately for the weather.

5. Carousel for Clothing Items
A carousel screen has been added to allow users to browse their clothing items in a visually appealing way. This feature displays images of clothing items in a carousel format, providing a smooth scrolling experience to navigate through the closet's inventory.

Users can swipe through clothing items to quickly see their wardrobe and select items they want to view or add to outfits.

6. CRUD Operations
The app continues to support CRUD operations for managing clothing items and outfits:

Create: Users can add new clothing items and outfits to their closet.

Read: Users can view clothing items, details, and outfits.

Update: Users can edit existing items or outfits.

Delete: Users can remove items or outfits from the app, ensuring their closet remains organized.

7. Navigation & Menus
The app includes easy-to-use navigation menus for seamless interaction between the Home Screen, Clothing Screen, Outfit Management, and other sections. Users can easily switch between screens and manage their clothing and outfits without getting lost.

Material Design Elements: Action Buttons for creating new clothing items and outfits.

8. Design & User Interface
The app follows a clean and modern design with a blue color scheme and material design principles.

Icons for favorites (heart icon) and search are added to improve the interactivity of the app.

Responsive Layouts for various screen sizes, ensuring a smooth experience on all devices.

9. UI Elements
A variety of UI elements have been added to enhance usability:

Date Picker: For users to select the last-worn date of an item.

Number Input: For specifying clothing size.

Multi-line Text Fields: For detailed descriptions or comments about clothing items.

Check-boxes: For choosing which clothing items make an outfit
